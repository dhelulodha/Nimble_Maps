About Nimble Maps

Nimble Maps
- An efficient but easy to understand Map.

This Project is about making the availabe google maps much more powerful by giving practical solutions to everyday problems which can be accesed by everyone. The project aims to integrate the google maps with the webapp and use machine learning model to solve the problems and make the model smarter.

AIM

Problem Statements

Potholes and Types: Provide better Locomotion & Real-Time tracking of road repairs.
Illumination level (Lux): Public Safety: Women Safety, Nightlife, Street lights working status.
Noise Level (dB): RealEstate solutions,enforcing Legal limits on noise-level at night.

Tracking of Potholes

Pothole Detection and Mapping

An API will be responsible for collecting Potholes and illumination data which will be taken by all the users with coordinate information(extracted by GPS). The validation of the data will be done and then it will be categorized based on pothole size. Each of which will be placed in the same GPS map application. The potholes information will be taken as a photo evidence with coordinate(latitude & longitude) information both of which will then be plotted on the map.

Illumination

Illumination Intensity (Lux)

An API will be responsible for illumination data which will be taken by all the users with coordinate information (extracted by GPS), cctv, dashcam, etc. The validation of the data will be done and then it will be categorized based on illumination level. Each of which will be placed in the same GPS map application. For illumination street lights and building lights will be usedwith coordinate (latitude & longitude) information both of which will then be plotted on the map.

Noise

Noise Level Mapping

Noise data from all over the city will be mapped and color coded. Then classification will be done based categories with the use of Machine Learning algorithms which will produce a color shade over the map (each shade pertains to a noise level). This will then be interfaced with any popular GPS map application to be used as a feature at users’ requirement.
